## 3. 비교에 사용한 동일 입력

### 업무 과업 프롬프트 (공통)

```
[업무 과업] 회의록 요약

[원문]
(아래 회의록 원문 붙여넣기)

[원하는 결과]
- 회의 개요 (일시·장소·목적)
- 참석자 (역할명 기준)
- 결정사항
- Action Items (담당자 역할명 / 작업 내용 / 기한)
- 리스크 및 이슈
- 후속 일정

[톤] 간결, 사내 공유용, 격식체

[금지]
- 추측성 표현 금지
- 인물 실명 대신 역할명 사용
- 원문에 없는 수치·날짜 임의 생성 금지
```

### 입력 원문 개요

| 항목 | 내용 |
|------|------|
| **회의명** | 2025년 하반기 신규 서비스 런칭 종합 점검 회의 |
| **일시** | 2025년 6월 12일(목) 14:00~16:10 (2시간 10분) |
| **참석자** | 10명 (사업기획팀장, PM, 백엔드 개발 리드, 프론트엔드 개발, UX 디자인 리드, 데이터 분석, 마케팅, QA 리드, 법무/컴플라이언스, CS운영) |
| **안건 수** | 7개 안건 |
| **핵심 수치** | 월 9,900원/연 99,000원, 프로모션가 4,900원, 외주비 1,200만원, QA 기기비 480만원, 마케팅 예산 8,500만원, AI 추천 정확도 72% 등 |

## 5. 모델별 결과 요약

### ChatGPT (GPT-5.3 Instant) — 87점

6개 섹션을 완벽히 준수하고 회의 개요·Action Items·후속 일정을 모두 표 형식으로 출력했다. 「플러스멤버십」 표기, AI 추천 정확도(72%) 미달 리스크, 마케팅 타임라인 세부 일정까지 포함해 완성도가 높았다. 단, 사업기획팀장 Action Item 기한을 "내일까지"로 표기해 날짜 형식 통일이 되지 않은 점과, 기본가격(월 9,900원·연 99,000원)·B2B 제외 결정이 결정사항에 누락된 점이 아쉬웠다.

### Gemini 2.0 Flash — 85점

B2B 요금제 제외·멤버십 배지 5종 구체 구성·GDPR 최소 대응 등 결정사항을 풍부하게 포함했다. Action Items 10개를 날짜 형식(2025년 6월 13일)으로 정확히 기재했으며, 마케팅 단계별 일정도 후속 일정에 포함했다. 단, 결정사항을 요청 형식(개별 항목)이 아닌 3개 그룹 불릿으로 재구성하여 세부 항목 가시성이 낮아졌고, 리스크 섹션이 3개로 압축되어 AI 추천 정확도 미달·간편결제 손실 우려 등 일부 리스크가 누락됐다.

### Perplexity Sonar — 72.5점

간편결제 제외 시 결제 비중 39% 손실 가능성·AI 추천 정확도 72% 미달 등 원문에 있는 중요 리스크 정보를 다른 모델보다 더 명시적으로 포함한 점은 강점이었다. 그러나 **Action Items를 표 형식이 아닌 불릿+슬래시 구분 방식**으로 출력하여 형식을 준수하지 못했고, 날짜 형식도 "익일까지"가 혼재했다. 외주 비용·기본가격·마케팅 예산 등 핵심 수치가 다수 누락되었다.

## 6. 최종 선정 결론

### 🥇 채택 모델: ChatGPT (GPT-5.3 Instant) — 87점

회의록 요약 과업에서 **형식 준수(19/20)**와 **문맥 유지(17/20)**가 가장 중요하다. ChatGPT는 요청한 6개 섹션을 완벽히 준수하고 표 형식을 정확히 활용하여 즉시 공유 가능한 완성도를 보였다. Gemini와의 점수 차이(87 vs 85)는 작지만, 형식 준수에서 2점 차이(19 vs 17)가 결정적이었다.

| 채택 근거 | 점수 | 내용 |
|----------|:----:|------|
| ✅ 형식 준수 | 19/20 | 6개 섹션 완벽, 표 3개(회의 개요·Action Items·후속 일정) 정확 활용 |
| ✅ 문맥 유지 | 17/20 | 7개 안건 균등 커버, 마케팅 타임라인 세부 포함 |
| ✅ 한국어 완성도 | 9/10 | 경영진 보고용 즉시 활용 가능한 격식체 |

**감수하는 단점:**

| 단점 | 판단 |
|------|------|
| 기본가격·B2B 제외 결정 미포함 | 시스템 프롬프트(v2+MIRA) 적용 시 개선 가능 |
| 날짜 형식 "내일까지" 잔존 | v2 프롬프트에서 해결 확인됨 |
