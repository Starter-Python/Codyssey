## 1. 비교 목적

"플러스멤버십 런칭 종합 점검 회의" 회의록을 입력으로, **회의록 요약 자동화** 과업에 가장 적합한 LLM 모델을 선정한다.
선정 기준은 "느낌"이 아닌 7개 평가 축의 점수와 근거로 한다.

> 핵심 과업 요구사항: 긴 문맥(2시간 10분, 7개 안건, 10명 참석자)을 균등하게 유지하고, 지정된 템플릿 형식을 정확히 따르며, 원문에 없는 정보를 생성하지 않는 능력

## 3. 비교에 사용한 동일 입력

세 모델에 동일한 회의록 원문과 동일한 유저 프롬프트를 입력하였다.

### 3-1. 유저 프롬프트 (공통)

```
[업무 과업] 아래 회의 내용을 요약해줘.


[원문]
```

### 3-2. 입력 원문 개요

| 항목 | 내용 |
|------|------|
| **회의명** | 2025년 하반기 신규 서비스 런칭 종합 점검 회의 |
| **일시** | 2025년 6월 12일(목) 14:00~16:10 (2시간 10분) |
| **장소** | 본사 12층 대회의실 + Zoom 병행 (하이브리드) |
| **참석자** | 10명 (사업기획팀장, PM, 백엔드 개발 리드, 프론트엔드 개발, UX 디자인 리드, 데이터 분석, 마케팅, QA 리드, 법무/컴플라이언스, CS 운영) |
| **안건 수** | 7개 안건 (프로덕트 기획 / 개발 현황 / AI 추천 / 마케팅 / QA 일정 / CS 운영 / 기타 논의) |
| **핵심 수치** | 가격 정책(9,900원/99,000원), 프로모션가(4,900원), 외주비(1,200만원), QA 기기비(480만원), 마케팅 예산(8,500만원) 등 |

> 원문 전체 분량 약 1,800자. 모델별 동일 원문 입력 확인.

## 5. 모델별 결과 요약

### Gemini 2.0 Flash

주요 결정사항은 4개 그룹(프로덕트/개발/데이터·AI/마케팅·QA·CS)으로 묶어 서술하였다. 핵심 기능·가격 정책·외주 투입 결정·A/B 테스트 승인 등 중요 결정사항을 포함하였으나, 액션아이템을 6개로 압축하여 담당자(강하은 CS 운영, 박정우 백엔드 개발 리드 등)의 Action Item이 누락되었다. 이모지 헤더(📌💡📋)를 사용하여 캐주얼한 톤이 혼재하였으며, "파트별 현황" 섹션과 "후속 일정 전체 표"가 없었다. 회의 후반부 안건(CS 운영 세부 사항, 인플루언서 계약 세부 등)의 정보 손실이 발생하였다.

### ChatGPT (GPT-5.3 Instant)

6개 섹션(회의 개요·결정사항·부문별 진행 현황·주요 일정·주요 Action Items·종합 평가)을 모두 포함하였다. 결정사항 6개, 부문별 현황(프로덕트·백엔드·프론트엔드·AI 추천·법무·마케팅·CS) 7개 파트, Action Item 8명 전원을 빠짐없이 기재하였다. 가격 정책(월 9,900원·연간 99,000원), B2B 제외 결정, 마케팅 예산 세부(SNS 3,000만원·인플루언서 2,500만원 등)까지 원문의 수치를 정확히 인용하였다. 격식체 비즈니스 톤이 일관되게 유지되었으며, 경영진 보고용으로 즉시 활용 가능한 완성도를 보였다. 단, 사업기획팀장 Action Item 기한을 "익일"로 표기하여 날짜 형식 불일치가 1건 발생하였다.

### Perplexity Sonar

자체 구조(핵심목표/결정사항/파트별/리스크/일정/한 줄 요약)로 재구성하여 가독성은 높았으나, 지정 템플릿 형식을 따르지 않았다. "한 줄 요약" 마무리 문장은 가독성 측면에서 좋은 구성이었다. 그러나 정상가(9,900원)·연간가(99,000원)·외주 비용(1,200만원) 등 핵심 수치를 누락하였고, 백엔드 세부 진행률(결제 40%·구독 70%·권한 80%)을 "70~80%"로 뭉뚱그렸다. 마케팅 예산 세부 항목도 생략되었다.

## 6. 최종 선정 결론

### 🥇 채택 모델: ChatGPT (GPT-5.3 Instant)

**채택 근거:**

"회의록 요약" 과업은 긴 문맥 유지와 형식 준수가 가장 중요한 능력이다. 회의록에는 다수의 안건·담당자·수치·기한이 혼재하므로, 후반부 정보까지 균등하게 유지하고 지정 템플릿을 충실히 따르는 능력이 핵심이다.

ChatGPT는 7개 안건 전체를 균등하게 커버하고(문맥 유지 18/20), 지정 템플릿 6개 섹션을 완벽히 반영하였으며(형식 준수 19/20), 기재한 모든 수치에서 환각이 발생하지 않았다(정확성 18/20). 특히 경영진 보고용으로 즉시 활용 가능한 완성도를 보였으며, 종합 점수 90점으로 타 모델(Gemini 79점, Perplexity 77점) 대비 확실한 우위를 보였다.

| 채택 근거 | 점수 | 내용 |
|----------|:----:|------|
| ✅ 문맥 유지 | 18/20 | 7개 안건 전체를 균등하게 커버 |
| ✅ 형식 준수 | 19/20 | 지정 템플릿 6개 섹션 완벽 반영 |
| ✅ 정확성 | 18/20 | 기재한 수치 전부 정확, 환각 없음 |
| ✅ 활용 완성도 | — | 경영진 보고용 즉시 사용 가능 |

**감수하는 단점:**

| 단점 | 판단 |
